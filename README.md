# 벨로그 클론코딩 과제

## 구현 범위

- 메인 페이지
- 상세 페이지

## 제공된 것 / 직접 해야 하는 것

**제공된 것 (건드리지 않아도 됨)**

- Vite + React(JS) 기본 세팅
- `src/data/dummy_data.js`
- `src/App.jsx` — 초기 진입 화면

**과제 범위 (TODO)**

- `react-router-dom` 설치 및 라우터 세팅
- UI 구현 : 메인 페이지, 상세 페이지 UI
- 페이지 이동 (라우팅)
- 더미데이터를 props로 연결

## 조건

### UI 조건
- [ ] 벨로그 메인 페이지 UI를 최대한 동일하게 구현한다 (F12 개발자모드로 실제 스타일 값 확인 가능)
- [ ] 벨로그 상세 페이지 UI를 최대한 동일하게 구현한다
- [ ] 컴포넌트 단위로 최대한 쪼개서 구현한 뒤 조립한다 (예: Header, PostCard, PostList 등)

### 라우팅 조건
- [ ] `/` 경로 → 메인 페이지
- [ ] `/posts/:id` 경로 → 상세 페이지 (`useParams`로 id를 읽어 해당 게시글을 찾아 렌더링)
- [ ] 메인 페이지의 게시글 카드를 클릭하면 해당 게시글의 상세 페이지로 이동한다

### 데이터 조건
- [ ] `src/data/dummy_data.js`의 배열을 `map`으로 순회해서 카드 목록을 렌더링한다
- [ ] 각 게시글 데이터는 하위 컴포넌트에 **props로 전달**한다 (ex. `<PostCard post={post} />` 형태)
- [ ] 상세 페이지도 동일한 더미데이터에서 `id`가 일치하는 게시글을 찾아 props로 내려서 렌더링한다

## 시작하기

```bash
npm install
npm run dev
```
